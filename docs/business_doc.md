# Voices-style Scanner Game (MVP) — бізнес-документ

## Короткий опис
Навчальний веб-застосунок про пошук і обробку “сигналів” із далекого космосу за допомогою віртуальної наземної обсерваторії.
Клієнт відображає “вікно неба”, де користувач робить ping, сканує та отримує сигнали. Бекенд надає REST API та працює з БД (SQLite; у майбутньому — Postgres).
Проєкт є базою для наступних лабораторних.

## Ролі користувачів
- **Operator / Гравець** — керує віртуальною апаратурою пошуку, запускає сканування, переглядає та обробляє знайдені сигнали.
- **Admin** — сервісна роль (на етапі MVP опційно): змінює параметри генерації/ймовірності подій для демонстрацій.

## Основні сценарії
1. Operator відкриває сторінку та натискає **Ping** → бачить доступні “маячки” на карті.
2. Обирає точку та натискає **Scan** → отримує результат (успіх/помилка) і запис у **History**.
3. Переглядає **History**, розгортає деталі конкретного сигналу.
4. (Далі в курсі) Переходить на панель **Polarity / Frequency** для поглибленої обробки.

## Функціональні вимоги
- Відображення “вікна неба” з можливістю ping і сканування.
- Збереження результатів скану в БД та перегляд історії на клієнті.
- Базовий health-check бекенду.

## Нефункціональні вимоги
- Локальний запуск за README (≤ кілька команд), час відповіді API < 1 c у типовому кейсі.
- Зрозумілий UI (мінімум кліків), робота в сучасних Chromium-браузерах.
- Валідація вхідних параметрів, відсутність секретів у репозиторії.

## Обмеження
- Фізика сигналів спрощена; це навчальна симуляція.
- Aутентифікація й авторизація відсутні на етапі MVP.

## Поза межами (заплановано надалі)
- Панель **Polarity / Frequency** із індикаторами “sweet-spot”.
- Велика карта неба з панорамуванням, ping-маячки, state-machine сканування.
- Перехід на Postgres, деплой прев’ю-версій, економіка ігрової валюти.
