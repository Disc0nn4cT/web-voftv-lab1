## Тиждень 1 - Документи та зміцнення MVP
- Завершити документи: `business_doc.md`, `architecture_diagram.png`.

## Тиждень 2 - Вікно неба та процес сканування
- Реалізувати **sky viewport/trackpad** із панорамуванням (2D «вікно у небо», без реальної астрономії).
- Додати **Ping (заглушка)**: фейковий список «маячків» (поки що генерується на клієнті).
- **Становий автомат сканування (на клієнті):**
  - Старт скану → показ прогресу/логу; на час скану блокувати керування.
  - Завершення: успіх/провал (30% провалу); запис результату в БД через `/signal?az&el`.
  - У DOM тримати лише N найновіших рядків логу (для продуктивності).

---

## Тиждень 3 - Прев’ю «Processing» та підготовка до наступних лаб
- Додати **панель Processing (прев’ю)** з двома індикаторами:
  - **Polarity preview** (радарна стрілка, прив’язана до числового значення).
  - **Frequency preview** (хвиля, прив’язана до числового значення).
- Навігація: кнопка перемикання **Scanner ↔ Processing**; стан зберігається в пам’яті сторінки.
- Підготувати опис майбутніх API (без реалізації на бекенді): в README перелічити ендпойнти Scan State та Processing.
